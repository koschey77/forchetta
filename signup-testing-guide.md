# 🎯 Схема сценариев регистрации пользователя

> **Полная карта возможных путей развития событий для пользователя в процессе регистрации**

---

## 📋 Общие параметры системы

- **⏰ Время на код**: 2 минуты (120 секунд)
- **🔢 Попытки ввода**: 3 попытки
- **🔄 Повторная отправка**: 1 раз за сессию
- **📧 Email шаблоны**: Единый дизайн Forchetta
- **🔐 Безопасность**: Автоматическая очистка данных

---

## 🛤️ КАРТА ВСЕХ СЦЕНАРИЕВ

### 🚀 **Путь А: Идеальная регистрация**

```
👤 Пользователь заходит на /signup
    ↓
📝 Вводит: name, email (новый), password
    ↓
✅ Нажимает "Зареєструватися"
    ↓
📧 Получает email с кодом (123456)
    ↓
🔄 Переход на /verify-email?email=user@example.com
    ↓
⏰ Таймер: 2:00 → 1:59 → 1:58...
    ↓
✅ Вводит правильный код: 123456
    ↓
🎉 УСПЕХ: Автовход + Welcome email + переход на /
```

    ↓

## 🎭 **Особые сценарии**

### 🔥 **Resend уже использован**
```
Если пользователь попытается вызвать resend второй раз:
❌ "Повторна відправка вже використана. Begin registration again."
```

### ⚡ **Сессия истекла во время resend**
```
Если TTL истек, а пользователь пытается resend:
❌ "Сессія верифікації істекла. Почніть реєстрацію заново."
```

### 🛡️ **MongoDB конфликт во время верификации**
```
Если между получением кода и его вводом кто-то зарегистрировался с тем же email:
❌ "Email вже існує. Почніть реєстрацію заново."
```

4. **🔄 Resend логика**: Однократность использования
5. **🛡️ Защита**: Невозможность обхода ограничений
6. **🎨 UI состояния**: Правильные кнопки в каждой ситуации
7. **📧 Email доставка**: Получение кодов в почте
8. **🍪 Sessions**: Сохранение состояния между страницами
9. **💥 Edge cases**: Истечение сессии, дублирование email
10. **📱 UX потоки**: Логичные переходы между страницами
    |

### 📧 ЭТАП 2: Страница верификации email (/verify-email)

```
🌐 Открываем /verify-email?email=newuser@example.com
|
|__📧 Email отображается из URL (неактивное поле)
|__⏰ Таймер обратного отсчета: 60 секунд
|__🖊️ Поле ввода 6-значного кода
|
|__✅ ТЕСТ 1: Правильный код в течение времени
|   |__📧 Берем код из email (например: 123456)
|   |__🖊️ Вводим код: 123456
|   |__🔘 Нажимаем "Підтвердити"
|   |__⏳ Показывается индикатор загрузки
|   |__✅ РЕЗУЛЬТАТ: Toast "Ласкаво просимо!" + автовход выполнен
|   |__➡️ Автоматический переход на главную страницу /
|   |__🔐 В системе: пользователь создан в MongoDB, JWT токены выданы
|   |__✅ ПРОВЕРКА: Пользователь авторизован, видно имя в интерфейсе
|
|__❌ ТЕСТ 2: Неправильный код (1-2 попытки)
|   |__🖊️ Вводим неверный код: 999999
|   |__🔘 Нажимаем "Підтвердити"
|   |__❌ РЕЗУЛЬТАТ: Красное сообщение "Невірний код. Залишилось спроб: 2"
|   |__⏰ Таймер сбрасывается на 60 секунд (новая минута!)
|   |__🔄 Повторяем с неверным кодом:
|   |   |__2-я попытка: "Залишилось спроб: 1" + новая минута
|   |__📊 В системе: счетчик попыток увеличивается, TTL продлевается
|
|__🔥 ТЕСТ 3: Превышение лимита попыток (3-я попытка)
|   |__🖊️ Вводим неверный код: 888888 (3-я попытка)
|   |__🔘 Нажимаем "Підтвердити"
|   |__❌ РЕЗУЛЬТАТ: Toast "Вичерпано кількість спроб"
|   |__🗑️ В системе: данные удалены из Redis
|   |__🔘 Появляется кнопка "Зареєструватися знову"
|   |__🔘 Нажимаем кнопку
|   |__➡️ Переход обратно на /signup
|   |__🔄 Нужно заново пройти весь процесс регистрации
|
|__⏰ ТЕСТ 4: Истечение времени (60 секунд)
|   |__📧 Получаем код из email
|   |__⏰ НЕ вводим код, ждем истечения таймера (60 сек)
|   |__⏰ РЕЗУЛЬТАТ: "Час минув" + кнопка "Почати знову" становится активной
|   |__🖊️ Пытаемся ввести правильный код из email
|   |__❌ Поле ввода неактивно или код не принимается
|   |__🔘 Нажимаем "Почати знову"
|   |__➡️ Переход на /signup
|   |__🗑️ В системе: данные автоматически удалены (TTL истек)
|
|__❌ ТЕСТ 5: Код истек между попытками
|   |__📧 Получаем код из email
|   |__🖊️ Вводим неверный код 2 раза (получаем продления времени)
|   |__⏰ На 3-й попытке ждем истечения времени
|   |__🖊️ Пытаемся ввести ПРАВИЛЬНЫЙ код из email
|   |__❌ РЕЗУЛЬТАТ: "Код підтвердження застарів"

|   |__🔘 Нажимаем "Підтвердити"
|   |__❌ РЕЗУЛЬТАТ: Toast с сетевой ошибкой
|   |__🔄 Восстанавливаем соединение, повторяем
|   |__✅ При наличии времени - должно работать

    |__🔘 Нажимаем

## 🎯 Дополнительные тесты безопасности

### 🛡️ Тестирование защиты системы

```
🔍 ТЕСТ: Попытка использования чужого кода
|__📧 Получаем код для user1@example.com
|__📧 На странице верификации подставляем user2@example.com
|__🖊️ Вводим код от user1
|__❌ РЕЗУЛЬТАТ: "Код підтвердження не знайдено або застарів"

🔍 ТЕСТ: Попытка повторного использования кода
|__📧 Получаем код, успешно регистрируемся
|__🔄 Пытаемся снова использовать тот же код
|__❌ РЕЗУЛЬТАТ: Код не найден (удален из Redis)
|__🛡️ ВЫВОД: Одноразовое использование кода

🔍 ТЕСТ: Пербор кодов
|__📧 Получаем процесс верификации для реального email
|__🔄 Пробуем коды: 000000, 000001, 000002...
|__⏰ ОЖИДАНИЕ: После 3 попыток процесс должен прерваться
|__❌ РЕЗУЛЬТАТ: "Вичерпано кількість спроб"
|__🛡️ ВЫВОД: Защита от брутфорса работает

🔍 ТЕСТ: Гонка условий (race conditions)
|__📧 Получаем код верификации
|__🔄 Отправляем ОДНОВРЕМЕННО несколько запросов с правильным кодом
|__✅ ОЖИДАНИЕ: Только один запрос должен создать пользователя
|__❌ Остальные должны получить ошибку "уже существует"
|__🛡️ ВЫВОД: Нет дублирования пользователей
```

## 🚨 Критические ошибки для немедленного исправления

❌ **Если найдены:**
- Возможность регистрации с одним email несколько раз
- Коды верификации не истекают
- Отсутствие лимитов попыток ввода кода
- Пароли сохраняются в открытом виде
- Возможность использовать чужой код верификации
- Токены аутентификации не выдаются после регистрации
- Временные данные не удаляются из Redis
- Отсутствие валидации входных данных

3. **Ожидаемый результат**

5. **Скриншот/видео** (если возможно)
