# 🔐 Инструкция для тестирования восстановления пароля

## 📋 Общая информация для тестировщика

**Что тестируем:** Полный функционал восстановления и сброса пароля  
**Временные лимиты:** 15 минут на ввод кода, 60 секунд защита от спама  
**Лимиты попыток:** 5 попыток ввода кода, 3 запроса в час  
**Страницы:** `/forgot-password` → `/reset-password` → `/login`

---

## 🧪 Подробная схема тестирования

### 📱 ЭТАП 1: Страница запроса восстановления (/forgot-password)

```
🌐 Открываем страницу /forgot-password
|
|__📧 Поле ввода email
    |
    |__✅ ТЕСТ 1: Правильный email существующего пользователя
    |   |__🖊️ Вводим: user@example.com (существует в системе)
    |   |__🔘 Нажимаем "Отправить код"
    |   |__⏳ Показывается индикатор загрузки
    |   |__✅ РЕЗУЛЬТАТ: Toast "Код відновлення відправлено на email"
    |   |__📧 Проверяем почту - должен прийти код (6 цифр)
    |   |__➡️ Автоматический переход на /reset-password?email=user@example.com
    |   |__⏰ В системе: код действует 15 минут, счетчик запросов +1
    |
    |__✅ ТЕСТ 2: Email несуществующего пользователя (проверка безопасности)
    |   |__🖊️ Вводим: nonexistent@example.com (НЕ существует)
    |   |__🔘 Нажимаем "Отправить код"  
    |   |__✅ РЕЗУЛЬТАТ: Toast "Код відновлення відправлено на email" (тот же!)
    |   |__📧 Проверяем почту - НИ ОДНОГО письма не приходит
    |   |__➡️ Переход на /reset-password?email=nonexistent@example.com
    |   |__🛡️ Безопасность: система не раскрывает существование email
    |
    |__❌ ТЕСТ 3: Неправильный формат email
    |   |__🖊️ Вводим: invalidemailformat
    |   |__🔘 Нажимаем "Отправить код"
    |   |__❌ РЕЗУЛЬТАТ: Поле подсвечивается красным
    |   |__📝 Исправляем на правильный формат: user@domain.com
    |   |__✅ Поле становится нормальным, можно отправить
    |
    |__🔥 ТЕСТ 4: Превышение лимита запросов (3 в час)
    |   |__🔄 ПОДГОТОВКА: Отправляем 3 запроса подряд с одним email
    |   |   |__1️⃣ Первый запрос: ✅ Успех
    |   |   |__2️⃣ Второй запрос: ✅ Успех  
    |   |   |__3️⃣ Третий запрос: ✅ Успех
    |   |__🔘 Четвертый запрос с тем же email
    |   |__❌ РЕЗУЛЬТАТ: Toast "Забагато спроб відновлення пароля. Спробуйте через годину"
    |   |__⏰ Ждем 1 час или используем другой email для продолжения тестов
    |
    |__❌ ТЕСТ 5: Сетевая ошибка
        |__🔌 ПОДГОТОВКА: Отключаем интернет или блокируем API
        |__🖊️ Вводим корректный email
        |__🔘 Нажимаем "Отправить код"
        |__❌ РЕЗУЛЬТАТ: Toast с текстом ошибки сети
        |__🔄 Восстанавливаем соединение, повторяем - должно работать
|
|__🔙 Кнопка "Назад к входу"
    |__🔘 Нажимаем
    |__➡️ РЕЗУЛЬТАТ: Переход на страницу /login
```

### 🔐 ЭТАП 2: Страница сброса пароля (/reset-password)

```
🌐 Открываем /reset-password?email=user@example.com
|
|__📧 Email отображается из URL (поле неактивно)
|__⏰ Таймер защиты от спама: 60 секунд (кнопка "Повторить" неактивна)
|__🖊️ Поле ввода 6-значного кода
|__🖊️ Поле ввода нового пароля  
|
|__✅ ТЕСТ 1: Правильный код + правильный пароль
|   |__📧 Берем код из email (например: 123456)
|   |__🖊️ Вводим код: 123456
|   |__🖊️ Вводим пароль: myNewPassword123 (>=6 символов)
|   |__🔘 Нажимаем "Сбросить пароль"
|   |__⏳ Показывается индикатор загрузки
|   |__✅ РЕЗУЛЬТАТ: Toast "Пароль успішно змінено!"  
|   |__➡️ Автоматический переход на /login
|   |__🔐 В системе: пароль изменен, все refresh токены аннулированы
|   |__✅ ПРОВЕРКА: Входим на /login с новым паролем - должно работать
|
|__❌ ТЕСТ 2: Неправильный код (1-4 попытки)
|   |__🖊️ Вводим неверный код: 999999
|   |__🖊️ Вводим правильный пароль: myNewPassword123
|   |__🔘 Нажимаем "Сбросить пароль"
|   |__❌ РЕЗУЛЬТАТ: Toast "Невірний код відновлення. Залишилось спроб: 4"
|   |__🔄 Повторяем с неверными кодами:
|   |   |__2-я попытка: "Залишилось спроб: 3"  
|   |   |__3-я попытка: "Залишилось спроб: 2"
|   |   |__4-я попытка: "Залишилось спроб: 1"
|   |__📊 В системе: счетчик попыток увеличивается, данные остаются в Redis
|
|__🔥 ТЕСТ 3: Превышение лимита попыток (5-я попытка)
|   |__🖊️ Вводим неверный код: 888888 (5-я попытка)
|   |__🖊️ Вводим правильный пароль: myNewPassword123
|   |__🔘 Нажимаем "Сбросить пароль"
|   |__❌ РЕЗУЛЬТАТ: Toast "Перевищено кількість спроб. Запросіть новий код відновлення"
|   |__🗑️ В системе: данные удалены из Redis, код аннулирован
|   |__➡️ Автоматический переход на /forgot-password
|   |__🔄 Нужно запросить новый код для продолжения
|
|__⏰ ТЕСТ 4: Истечение времени кода (15 минут)
|   |__📧 Получаем код по email
|   |__⏰ ЖДЕМ 16 минут (или меняем время в системе)
|   |__🖊️ Вводим правильный код из email
|   |__🖊️ Вводим правильный пароль: myNewPassword123
|   |__🔘 Нажимаем "Сбросить пароль"
|   |__❌ РЕЗУЛЬТАТ: Toast "Код відновлення застарів або не знайдений"
|   |__🗑️ В системе: данные автоматически удалены из Redis (TTL)
|   |__🔄 Нужно запросить новый код
|
|__❌ ТЕСТ 5: Слишком короткий пароль
|   |__🖊️ Вводим правильный код из email
|   |__🖊️ Вводим короткий пароль: 12345 (5 символов)
|   |__🔘 Нажимаем "Сбросить пароль"
|   |__❌ РЕЗУЛЬТАТ: Подсветка поля красным + сообщение "Пароль повинен містити мінімум 6 символів"
|   |__📝 Исправляем на пароль >=6 символов
|   |__✅ Поле становится нормальным
|
|__🔄 ТЕСТ 6: Повторная отправка кода
|   |__⏰ ЖДЕМ окончания таймера (60 секунд)
|   |__✅ Кнопка "Повторно отправить код" становится активной
|   |__🔘 Нажимаем "Повторно отправить код"
|   |__✅ РЕЗУЛЬТАТ: Toast "Код відновлення відправлено на email"
|   |__📧 Проверяем почту - приходит НОВЫЙ код
|   |__🗑️ Поле ввода кода очищается автоматически
|   |__⏰ Таймер сбрасывается на 60 секунд
|   |__🔢 В системе: старый код заменен новым, счетчик попыток обнулен
|
|__🔌 ТЕСТ 7: Потеря соединения 
|   |__🖊️ Заполняем правильные данные
|   |__🔌 Отключаем интернет
|   |__🔘 Нажимаем "Сбросить пароль"
|   |__❌ РЕЗУЛЬТАТ: Toast с ошибкой сети
|   |__🔄 Восстанавливаем соединение, повторяем - должно работать
|
|__🔙 Кнопка "Назад к запросу восстановления"
    |__🔘 Нажимаем
    |__➡️ РЕЗУЛЬТАТ: Переход на /forgot-password
```

### 🏁 ЭТАП 3: Финальная проверка входа

```
🌐 После успешного сброса пароля - переходим на /login
|
|__✅ ТЕСТ 1: Вход со старым паролем (должен НЕ работать)
|   |__🖊️ Вводим email: user@example.com
|   |__🖊️ Вводим СТАРЫЙ пароль: oldPassword123
|   |__🔘 Нажимаем "Войти"
|   |__❌ РЕЗУЛЬТАТ: "Неверный пароль" или "Неверные данные"
|   |__✅ ПРОВЕРКА ПРОЙДЕНА: старый пароль деактивирован
|
|__✅ ТЕСТ 2: Вход с новым паролем (должен работать)
|   |__🖊️ Вводим email: user@example.com  
|   |__🖊️ Вводим НОВЫЙ пароль: myNewPassword123
|   |__🔘 Нажимаем "Войти"
|   |__✅ РЕЗУЛЬТАТ: Успешный вход, переход на главную страницу
|   |__🔐 В системе: генерируются новые access/refresh токены
|   |__✅ ПРОВЕРКА ПРОЙДЕНА: новый пароль работает
```

## 🎯 Дополнительные тесты безопасности

### 🛡️ Тестирование защиты от атак

```
🔍 ТЕСТ: Попытка перебора email-адресов
|__🔄 Отправляем запросы с 10+ разными несуществующими email
|__✅ РЕЗУЛЬТАТ: Все получают одинаковый ответ "код отправлен"
|__📧 ПРОВЕРКА: НИ ОДНО письмо не приходит
|__🛡️ ВЫВОД: Система не раскрывает существование учетных записей

🔍 ТЕСТ: Попытка перебора кодов
|__📧 Получаем настоящий код для существующего пользователя
|__🔄 Пробуем коды: 000000, 000001, 000002... (автоматизированно)
|__⏰ ОЖИДАНИЕ: После 5 неверных попыток код должен аннулироваться
|__❌ РЕЗУЛЬТАТ: "Превышено количество попыток"
|__🛡️ ВЫВОД: Защита от брутфорса работает

🔍 ТЕСТ: Попытка использования истекшего кода
|__📧 Получаем код
|__⏳ Ждем 16+ минут
|__🖊️ Пытаемся использовать код
|__❌ РЕЗУЛЬТАТ: "Код застарів або не знайдений"
|__🛡️ ВЫВОД: TTL работает корректно
```

## 🚨 Критические ошибки для немедленного исправления

❌ **Если найдены:**
- Возможность узнать существование email по разным ответам
- Отсутствие лимитов попыток ввода кода
- Коды не истекают через 15 минут
- Возможность использовать старый пароль после сброса
- Отсутствие валидации пароля
- Коды не аннулируются при превышении попыток
- Нет защиты от спама (лимит запросов в час)

## 📞 Контакты для сообщения об ошибках

Найденные баги отправляйте с указанием:
1. **Шаг тестирования** (номер теста)
2. **Ожидаемый результат** 
3. **Фактический результат**
4. **Email для воспроизведения** 
5. **Скриншот/видео** (если возможно)
