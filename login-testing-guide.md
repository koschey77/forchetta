# 🔐 Инструкция для тестирования входа пользователя

## 📋 Общая информация для тестировщика

**Что тестируем:** Полный функционал входа в систему и управления сессией  
**Токены:** Access токен (15 минут), Refresh токен (7 дней)  
**Cookies:** HttpOnly cookies для безопасности  
**Страницы:** `/login` → `/` (главная) или `/logout`

---

    |__🔐 ПОДГОТОВКА: авторизуемся

    |__⏰ Ждем истечения access токена + делаем запрос
    |__❌ РЕЗУЛЬТАТ: токен не найден в Redis
    |__🚫 Автоматический выход
    |__🛡️ БЕЗОПАСНОСТЬ: централизованная отзывка токенов работает

### 🎯 Дополнительные тесты безопасности

```
🛡️ ТЕСТ 1: Защита от XSS (Cross-Site Scripting)
|__🔍 Пытаемся получить токены через JavaScript в консоли:
|   |__💻 document.cookie → токены недоступны (httpOnly)
|   |__💻 localStorage/sessionStorage → токены там не хранятся
|   |__✅ ВЫВОД: защита от XSS работает

🛡️ ТЕСТ 2: Защита от CSRF (Cross-Site Request Forgery)
|__🔍 Проверяем настройки cookies:
|   |__🍪 sameSite=strict → защита от межсайтовых запросов
|   |__🔒 secure=true (в production) → только HTTPS
|   |__✅ ВЫВОД: защита от CSRF настроена

🛡️ ТЕСТ 3: Попытка повторного использования старых токенов
|__🔐 Авторизуемся, получаем токены
|__🚪 Выходим (токены должны быть отозваны)
|__🛠️ Пытаемся подставить сохраненные токены в новой сессии
|__❌ РЕЗУЛЬТАТ: "Invalid refresh token" или автоматический выход
|__✅ ВЫВОД: отзыв токенов работает корректно

🛡️ ТЕСТ 4: Попытка входа с заблокированным аккаунтом
|__👤 ПОДГОТОВКА: создаем тестовый аккаунт, блокируем его
|__🖊️ Пытаемся войти с правильными credentials
|__❌ РЕЗУЛЬТАТ: "Account is blocked" или аналогичное сообщение
|__🚫 Токены не выдаются
|__✅ ВЫВОД: система предотвращает вход заблокированных пользователей

🔍 ТЕСТ 5: Одновременные сессии
|__🔐 Авторизуемся в браузере A
|__🔐 Авторизуемся в браузере B с теми же credentials
|__✅ РЕЗУЛЬТАТ: обе сессии должны работать независимо
|__🎫 У каждой сессии свои уникальные токены
|__✅ ВЫВОД: поддержка множественных сессий работает
```

## 🚨 Критические ошибки для немедленного исправления

❌ **Если найдены:**
- Токены доступны через JavaScript (не httpOnly)
- Пароли хранятся в открытом виде в базе данных
- Отсутствие валидации входных данных
- Токены не отзываются при выходе
- Возможность доступа к защищенным роутам без авторизации
- Токены не истекают или имеют слишком большое время жизни
- Сообщения об ошибках раскрывают существование учетных записей
- Отсутствие защиты от CSRF или XSS
- Возможность повторного использования отозванных токенов
- Нет автоматического обновления access токенов
